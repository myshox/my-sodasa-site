## 🔥 **高優先級建議（強烈推薦）**

### **1. 密碼安全性升級** 🔒
**嚴重性：高 | 影響：安全性 | 難度：中**

#### 問題：
- `admins` 表的密碼仍以明文儲存
- 存在重大安全風險

#### 解決方案 A（推薦）：完全遷移到 Supabase Auth
```sql
-- 將所有管理員遷移到 Supabase Auth
-- 刪除舊的 admins 表
-- 統一使用 user_metadata 儲存角色
```

**優點：**
- ✅ 使用業界標準的加密方式
- ✅ 內建密碼重設、MFA 等功能
- ✅ 統一認證系統

**缺點：**
- ❌ 需要遷移現有管理員帳號
- ❌ 需要修改登入邏輯

#### 解決方案 B（快速）：使用 bcrypt.js 加密
```javascript
// 前端使用 bcrypt.js 加密
import bcrypt from 'https://cdn.jsdelivr.net/npm/bcryptjs@2.4.3/+esm';

// 註冊時
const hashedPassword = await bcrypt.hash(password, 10);

// 登入時
const isValid = await bcrypt.compare(password, storedHash);
```

**優點：**
- ✅ 快速實作
- ✅ 不需要大幅修改架構

**缺點：**
- ❌ 前端加密仍有風險
- ❌ 需要修改現有密碼

2. **即時通知（Realtime）**
   ```javascript
   // 使用 Supabase Realtime
   supabase
     .channel('donations')
     .on('postgres_changes', {
       event: 'INSERT',
       schema: 'public',
       table: 'donations'
     }, payload => {
       showToast('新的贊助訂單！', 'success');
     })
     .subscribe();
   ```

3. **LINE Notify**（可選）
