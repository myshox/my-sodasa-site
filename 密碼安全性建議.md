# 🔐 密碼安全性改善建議

## ⚠️ 目前安全性問題

### **問題 1：明文密碼儲存**
- **現況**：`admins` 表中的密碼以明文形式儲存
- **風險**：資料庫若被洩露，所有管理員密碼將完全暴露
- **影響**：高風險

### **問題 2：雙重認證系統**
- **現況**：同時使用 `admins` 表（明文密碼）和 Supabase Auth（安全）
- **風險**：管理複雜度增加，容易產生安全漏洞
- **影響**：中風險

---

## ✅ 建議解決方案

### **方案一：完全遷移至 Supabase Auth（強烈推薦）**

#### **優點**
✅ 完全託管的認證系統，安全性由 Supabase 保證  
✅ 內建密碼加密（bcrypt）、會話管理、密碼重設  
✅ 支援多因素認證（MFA）、OAuth 等進階功能  
✅ 減少自行維護認證系統的負擔  
✅ 符合現代 Web 應用安全標準  

#### **實作步驟**

1. **為現有管理員建立 Supabase Auth 帳號**
   ```sql
   -- 讓現有管理員註冊 Supabase 帳號
   -- 然後執行此 SQL 設定他們的角色
   UPDATE auth.users 
   SET raw_user_meta_data = jsonb_set(
       COALESCE(raw_user_meta_data, '{}'::jsonb), 
       '{role}', 
       '"admin"'
   )
   WHERE email = 'admin@example.com';
   ```

2. **修改 AdminPage 登入邏輯**
   - 移除 `admins` 表登入
   - 改用 Supabase Auth (`supabase.auth.signInWithPassword`)
   - 檢查 `user.user_metadata.role` 來判斷權限

3. **（可選）保留 admins 表作為角色管理**
   - 移除 `password` 欄位
   - 只保留 `username`, `role`, `email` 等資訊
   - 透過 `email` 與 Supabase Auth 關聯

4. **優點總結**
   - ✅ 密碼永遠不會以明文儲存
   - ✅ 自動處理密碼重設、Email 驗證
   - ✅ 支援更多認證方式（Google, GitHub 等）
   - ✅ 內建 Session 管理和 Token 刷新

### **方案二：為 admins 表加入密碼加密（次選）**

如果無法立即遷移至 Supabase Auth，可以先加密現有密碼。

#### **實作步驟**

1. **前端加密（僅作為過渡方案）**
   ```javascript
   // 使用 crypto-js 或 bcryptjs
   import bcrypt from 'bcryptjs';
   
   // 註冊時
   const hashedPassword = bcrypt.hashSync(password, 10);
   
   // 登入時
   const isValid = bcrypt.compareSync(inputPassword, storedHashedPassword);
   ```

2. **⚠️ 重要警告**
   - 前端加密**不是真正的安全**（因為前端代碼可被檢視）
   - 只能防止資料庫洩露時密碼被直接看到
   - **無法防止中間人攻擊**或前端代碼被竄改
   - **強烈建議使用方案一（Supabase Auth）**

3. **遷移現有密碼**
   ```javascript
   // 一次性腳本：將所有明文密碼改為加密
   const admins = await supabase.from('admins').select('*');
   
   for (const admin of admins.data) {
       const hashedPassword = bcrypt.hashSync(admin.password, 10);
       await supabase
           .from('admins')
           .update({ password: hashedPassword })
           .eq('id', admin.id);
   }
   ```

## 📋 快速參考：Supabase Auth 登入範例

### **登入**
```javascript
const handleLogin = async (email, password) => {
    const { data, error } = await supabase.auth.signInWithPassword({
        email: email,
        password: password
    });
    
    if (error) {
        showToast('登入失敗：' + error.message, 'error');
        return;
    }
    
    // 檢查是否為管理員
    const user = data.user;
    const role = user?.user_metadata?.role;
    
    if (role !== 'admin' && role !== 'super_admin') {
        await supabase.auth.signOut();
        showToast('此帳號無管理員權限', 'error');
        return;
    }
    
    await logAction('login', 'admin', user.id, `管理員 ${user.email} 登入系統`);
    showToast('登入成功', 'success');
};
```

### **登出**
```javascript
const handleLogout = async () => {
    const user = await authHelpers.getCurrentUser();
    if (user) {
        await logAction('logout', 'admin', user.id, `管理員 ${user.email} 登出系統`);
    }
    
    await supabase.auth.signOut();
    showToast('已登出', 'success');
};
```

### **檢查登入狀態**
```javascript
useEffect(() => {
    const checkAuth = async () => {
        const { data: { user } } = await supabase.auth.getUser();
        
        if (!user || (!user.user_metadata?.role?.includes('admin'))) {
            // 未登入或非管理員
            setIsLoggedIn(false);
        } else {
            setIsLoggedIn(true);
            setCurrentUser(user);
        }
    };
    
    checkAuth();
    
    // 監聽認證狀態變化
    const { data: authListener } = supabase.auth.onAuthStateChange((event, session) => {
        if (event === 'SIGNED_OUT') {
            setIsLoggedIn(false);
        }
    });
    
    return () => {
        authListener.subscription.unsubscribe();
    };
}, []);
```
