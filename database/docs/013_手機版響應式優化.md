# 註冊用戶手機版響應式優化

## 更新內容

優化「註冊用戶」頁面的手機版顯示，提供更好的移動端用戶體驗。

## 優化重點

### 1. 響應式布局策略
- **桌面版（lg+）**：保持原有表格布局，適合大屏查看大量資料
- **手機版（<lg）**：改用卡片式布局，更適合小螢幕滑動瀏覽

### 2. 統計卡片優化
**手機版調整**：
- 改為 2 列布局（`grid-cols-2`）
- 平板 3 列（`md:grid-cols-3`）
- 桌面 5 列（`lg:grid-cols-5`）
- 減小內邊距（`p-3` → `md:p-4`）
- 縮小圖標（20px）和文字（`text-xs`）
- 最後一張卡片（累計儲值）佔滿 2 列

**優化細節**：
- 使用 `flex-shrink-0` 防止圖標被壓縮
- 使用 `min-w-0` 和 `truncate` 防止文字溢出
- 簡化文字（「超級管理員」→「超管」，「一般玩家」→「玩家」）

### 3. 手機版卡片設計

#### 卡片結構
```
┌─────────────────────────────────┐
│ [頭像] Email                     │
│        註冊日期                  │
├─────────────────────────────────┤
│ 🟣 超級管理員 / 🟡 管理員        │
├─────────────────────────────────┤
│ 累計儲值                         │
│ 1,500 元          🪙 15,000     │
├─────────────────────────────────┤
│ [操作按鈕]                       │
└─────────────────────────────────┘
```

#### 卡片特點
- 清晰的區塊分隔
- 重要資訊優先顯示
- 操作按鈕明確可觸控
- 適合單手操作

#### Email 區域
- 頭像：12x12，圓形，漸變背景
- Email：可換行顯示完整（`break-all`）
- 註冊日期：簡化顯示（只顯示日期）

#### 角色標籤
- 添加表情符號（🟣/🟡/⚪）增加辨識度
- 較大的內邊距（`py-1.5`）便於閱讀

#### 累計儲值區塊
- 深色背景區塊（`bg-stone-900/50`）突出顯示
- 金額左側，大字體（`text-2xl`）
- 金幣右側，帶背景色塊
- 橫向排列，視覺平衡

#### 操作按鈕
- **本人**：顯示提示文字
- **超管**：單個按鈕「降為管理員」
- **管理員**：2 個按鈕並排（升為超管 / 移除權限）
- **一般玩家**：2 個按鈕並排（設為管理員 / 設為超管）
- 使用 `grid-cols-2` 確保按鈕對齊
- 較大的觸控區域（`py-3`）

### 4. 響應式斷點

```
手機：< 768px  → 卡片布局，統計 2 列
平板：768-1024px → 卡片布局，統計 3 列
桌面：> 1024px → 表格布局，統計 5 列
```

## 技術實現

### CSS 工具類使用
```jsx
// 桌面顯示，手機隱藏
className="hidden lg:block"

// 手機顯示，桌面隱藏
className="lg:hidden"

// 響應式網格
className="grid grid-cols-2 md:grid-cols-3 lg:grid-cols-5"

// 響應式尺寸
className="text-xs md:text-sm"
className="p-3 md:p-4"
className="w-10 h-10 md:w-12 md:h-12"

// 防止溢出
className="min-w-0 truncate"
className="break-all"
```

### 布局邏輯
```jsx
{allUsers.length === 0 ? (
    // 空狀態
) : (
    <>
        {/* 桌面版：表格 */}
        <div className="hidden lg:block">...</div>
        
        {/* 手機版：卡片 */}
        <div className="lg:hidden">...</div>
    </>
)}
```

## 用戶體驗改善

### 手機版優勢
1. **更好的可讀性**：卡片布局更適合小螢幕
2. **更易操作**：按鈕更大，觸控友好
3. **資訊清晰**：重要資訊優先顯示
4. **滑動流暢**：垂直滾動，符合手機操作習慣
5. **無需橫向滾動**：避免表格橫向滾動的不便

### 桌面版優勢
1. **資訊密度高**：表格可同時顯示更多用戶
2. **易於比對**：橫向排列便於比較
3. **快速掃描**：適合管理大量資料

## 測試建議

### 手機測試
- 在瀏覽器開發者工具切換到手機模式
- 測試不同螢幕尺寸（iPhone SE, iPhone 12, Pixel）
- 確認按鈕觸控範圍足夠大
- 測試長 Email 的顯示

### 平板測試
- iPad / Android 平板
- 確認統計卡片為 3 列布局
- 測試橫向/直向模式

### 桌面測試
- 確認表格布局正常
- 測試統計卡片為 5 列布局

## 注意事項

- 手機版和桌面版使用相同的資料和功能
- 操作按鈕邏輯完全一致
- 確認對話框在手機上顯示正常
- Toast 提示訊息在手機上可見
